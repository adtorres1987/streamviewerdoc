The user is building SyncPDF from scratch using a pre-written architecture spec (`backend/syncpdf-backend.md`). They are implementing the backend in phases and expect production-quality code that faithfully follows the spec.

They write detailed task briefs with explicit constraints (what to build, what NOT to build). Follow the phase boundaries strictly — do not implement out-of-scope modules proactively.

The spec is the source of truth. Always read it before implementing anything new.
